## 还可以重点补做/增强的功能（按产品价值排序）

### 1. 从“库”到“复习系统”：加入复习节奏与学习闭环（最高优先级）
目前你有统计和筛选，但没有“复习触发/到期/完成反馈”。对学生来说，最缺的是“我今天该复习哪些错题？”这会直接决定留存与学习效果。

建议新增的能力：
- 每条错题的复习状态字段（产品层面需要的概念）
  - `dueAt`（下次复习时间）、`reviewCount`、`lastReviewedAt`
  - 复习结果（如：`again / hard / good / easy` 或类似评级）
- “每日复习/开始复习”入口
  - 从所有错题里挑出 `dueAt <= now` 的卡片
  - 复习时的 UI 流程：先看题图/标签，再逐步展示 notes/解析，最后让学生打分
- “复习后自动更新下一次到期时间”
  - 即便暂时不用完整算法，也可以先做简化规则（例如好→间隔延长，再次→间隔重置）

落点：把 `DuoNav` 的 `Library` 分成“Library + Review”，或者在 `Library` 增加一个“今日复习”模块（从信息架构上拉开学习优先级）。

### 2. 标签体系的进阶：从“能打标签”到“能管理标签”
你现在允许自定义标签，但缺少管理与质量控制，这会导致长期积累后出现同义词/拼写差异，影响统计与复习效率。

建议新增：
- 标签建议/自动补全：当输入自定义标签时，基于用户现有标签做 suggest
- 标签重命名/合并（可选）：合并同义标签后自动迁移错题标签
- 标签管理页（单独页面或弹窗）
  - 查看某标签下的数量趋势
  - 删除低用量标签（或提示该标签会影响历史统计）

参考现状：`/add` 和 `/library` 都是“自由输入 tags”，没有集中治理入口。

### 3. 统计从“Top 10”升级到“趋势与洞察”
当前 `TagStatsChart` 只做 Top 10 数量，适合入门，但对学习指导不够。

建议新增统计维度（产品上很有体感）：
- 时间趋势：过去 7/30 天新增错题数量、复习完成率
- 按标签的“增长/下降”趋势（例如哪个知识点错得越来越多）
- 复习效果：按标签看“被再次标记 again 的比例”（能定位薄弱点）
- 个性化仪表盘：比如“今日完成了 N 条复习”，“本周最常出错知识点”

### 4. 让“大量错题”依旧好用：排序、分页、批量操作
当错题数量变大后，当前“全量加载 + 单一排序（按创建时间 desc）”会变慢，筛选也会变得费劲。

建议新增：
- 排序选项（UI）：最新/最常错/最近编辑/最近复习
- 分页或无限滚动（产品上减少一次性渲染压力）
- 批量操作：
  - 多选卡片 → 批量添加/移除标签
  - 批量删除
  - 批量导出（用于备份）

### 5. 搜索能力增强：从“关键字匹配”到“可读的检索”
你现在搜索是 notes/tag 的简单包含匹配（`/library` 里实现）。可以明显增强“检索体验”。

建议新增：
- 高亮命中片段
- 更多筛选条件：日期范围、是否有 notes、是否包含某 preset 标签
- （进阶）OCR：从图片识别题干/公式，将“解析/题干”也纳入可检索内容
  - 这会极大提升“找回以前的错误”的速度

### 6. 录入页的体验细节：减少摩擦，让“拍照-保存”更顺滑
目前 `/add` 体验已经很好（预览、压缩、错误提示），但还可以再降低用户操作成本：

建议新增：
- 支持拖拽上传、粘贴图片（PC场景常用）
- 上传/保存过程的进度状态（尤其网络慢时）
- 保存成功后的轻提示 + “继续添加下一条”（而不是只跳转）
- 可选：图片裁剪/旋转（减少拍歪导致的后续阅读成本）

### 7. 账号与数据安全（中优先级）
现在有登录/注册，但“找回/备份/导出”对学生非常关键。

建议新增：
- 数据导出（zip：图片 + JSON/CSV：notes/tags）
- 账号删除（如果产品需要合规）
- 密码找回（如果后续面向真实校园上线）
